# is120-project2-Spencer-Kimball
Spencer Conference Website!

Questions to Answer:

**Q. In your README.md, include a brief description of your project and what you learned while building it:**

A. My project consists of the landing page for my Semi-annual Spencer Con group date event. I have been hosting this event for the last 4-5 years and have loved getting together
with Spencers around the Provo/Utah County area. Our group chat is now more than 100 Spencers in the surrounding area. I have been presenting a re-hashed slide show at each of
our events, but figured it was time to transition the content to a website thanks to this project. Since we have had 9 different events the time line is a little long, so I had
to include some buttons to try and help the flow of navigating the history.html page. 


**Q. I would love to hear about your initial ideas for approaching Project 2, problems you ran into, and how your project evolved:**
 
A. I wasn't sure if I wanted the time line to each be a card which then was a link to a separate html file for each of the events, but that would have meant creating
a bunch of different pages, and then if I were to change the header or footer, then that means I would have to change 9 different pages for each event that happens.
Then going forward when we have more events I would have to again create another html page. That didn't seem very sustainable to me, so going with just two html pages
seemed a lot more manageable. The first one index.html is just sharing the information about how to get involved and connected with the Spencers, and the second would
then be a very long page of the content of pictures and videos we have taken throughout Spencer Con.
 
**Q. Give descriptions for design and layout decisions that you made.**

A. I'm still not 100% sold on how my buttons look on the history.html page, but I do like the general feel for the page. Starting with the iPhone SE was the main formatting
at the beginning and then using media queries to adjust sizing of images and headers as the screen got larger. This was a much more seamless process than trying to go back and fix it after it had been built with only a desktop computer size in mind.

**Describe what tools you used in your project from the Dev Tool Demos and why?**

**My Dev Tool Demo Usage**
I used animistra for my on load animations to have the cards flip in, spin the home Scon icon in, bounce in the title, and slide in the navigation buttons on the history.html page with an accordian feel to it. I also used font pairing for my cards and my timeline headers and content.

**Classmate Dev Tool Demo Usage**
I used coolors to get my palette using the lock feature to lock in the colors that I liked. 
I also added some neumorphism to my buttons, which I like for the web version, but since you don't super see it on the mobile version I don't think it will be used that often since I bet my user groups (if they actually ever use this website) will only pull it up on their phones.

I double checked to make sure there were no accessibility errors and on wave there were no red alerts. There were some orange alerts, but nothing that really had to be changed.
